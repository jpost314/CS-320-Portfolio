# CS-320-Portfolio
- How can I ensure that my code, program, or software is functional and secure?
The only way to ensure that my code is functional and secure is via thorough testing developed around the system requirements. We do this by writing exhaustive unit tests, inspecting each line manually, and writing tests for edge cases. 

- How do I interpret user needs and incorporate them into a program?
Gathering requirements demands comprehensive communication with the stakeholders and thorough documentation. These requirements are then translated into smaller, acheivable goals for the development team. 

- How do I approach designing software?
Software design begins with gathering and understanding the stakeholders' requirements. After this initial gathering and documentation phase, the requirements are translated into individual goals. A blueprint is then created via UML diagrams. Then the actual coding and testing phase begins. All of these steps revolve around the requirements though.
